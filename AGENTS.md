# Repository Guidelines

## Project Structure & Module Organization

This is a Rust 2024 Cargo project for the `fmi_get_temp` CLI. `src/lib.rs` exports FMI fetching and MQTT/CoAP publishing functions. CLI parsing and runtime startup live in `src/bin/fmi_get_temp.rs`. Configuration flags are in `src/config.rs`, and HTTP helper code is in `src/web_util.rs`.

There is no dedicated `tests/` directory yet. Add integration tests there for cross-module behavior, and place small unit tests next to code under `#[cfg(test)]` modules.

## Build, Test, and Development Commands

- `cargo build`: compile the debug build.
- `cargo build --release`: compile the optimized release binary.
- `cargo build --profile minsize`: compile the stripped size-optimized binary.
- `cargo run -- --help`: inspect CLI options generated by `clap`.
- `cargo run -- --fmi-sid 101118`: fetch the latest temperature for the default Pirkkala station.
- `cargo test`: run unit and integration tests.
- `cargo fmt`: format code using `rustfmt.toml`.
- `cargo clippy --all-targets --all-features`: run lint checks before submitting changes.
- `cargo outdated --workspace --root-deps-only`: check root dependency freshness.

Networked runs call FMI Open Data by default. MQTT and CoAP publishing are opt-in via `--mqtt-enabled` and `--coap-enabled`.

## Coding Style & Naming Conventions

Use Rust naming: `snake_case` for functions, modules, variables, and CLI fields; `PascalCase` for types; `SCREAMING_SNAKE_CASE` for constants. Keep modules aligned with current responsibilities: configuration in `config`, HTTP utilities in `web_util`, and orchestration in `lib`.

Formatting uses `rustfmt.toml` with max line width 120. Its import grouping settings require nightly rustfmt and are ignored by stable.

## Testing Guidelines

Prefer deterministic tests that do not require live FMI, MQTT, or CoAP services. For parsing logic, use fixture XML strings. For network behavior, isolate URL construction and response handling. Name integration tests after the behavior they cover, for example `tests/fetch_temperature.rs`.

Run `cargo test` before opening a pull request. For changes to CLI flags, also verify `cargo run -- --help`.

## Commit & Pull Request Guidelines

Recent history uses short, lower-case commit subjects such as `cargo update`. Keep commit messages concise and imperative, for example `add mqtt publish test` or `update dependencies`.

Pull requests should include a brief description, commands run, and behavior changes. Link issues when applicable. Include sample output when changing CLI text or options.

## Security & Configuration Tips

Do not commit MQTT credentials or local endpoint secrets. Pass credentials with CLI flags or wrappers. Be cautious when changing TLS behavior in `src/web_util.rs`, especially certificate validation and timeouts.
